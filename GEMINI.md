# Project Conventions & Guidelines

## Versioning System
The project uses a custom versioning format: `v[Major].[Month].[Day]`
- **[Major]**: The major version number (e.g., `v4`).
- **[Month]**: The month the version was developed/released (e.g., `.5` for May).
- **[Day]**: The date of the last update/development (e.g., `.9` for the 9th).
Example: `v4.5.9` means Version 4, developed in May, last updated on the 9th.
